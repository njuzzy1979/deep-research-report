## 问题清单

### HIGH-1【矛盾残留】图片尺寸计算路径：01 与 02/04 给出互斥规格，R1-R12 未裁决
- **涉及**：01 §8.2（图片处理库行）、01 §4.3 FigureIR 注释 vs 02 §A.5 vs 04 §2.7
- **描述**：01 明确写"**不引入 Pillow：不读像素/DPI**，只传显式 width，python-docx 按图片原生纵横比自动算高度"，且 FigureIR 注释"无 width/height 字段"。02 §A.5 却要求"读取 PNG IHDR 像素尺寸…width 与 height **双参数显式传入**"，并给出 IMG_MAX_H_CM=20 高度上限和 W-IMG-02/I-IMG-03 分辨率警告——这三项功能**全部依赖读取像素宽高**，在 01 的"不读像素"约束下物理上无法实现。04 §2.7 同时引用了 01（"原生 API"）和 02（"显式双写宽高"），两头都采。05 号裁决备忘录对此零覆盖。
- **触发场景**：实现者按 01 实现 → 02 的 20cm 高度上限、W-IMG-02 警告成为死条款；按 02 实现 → 违反 01 的 FigureIR schema 与"不读像素"负面清单，code review 按 01 打回。附带一点：02 给出的双写理由（"只传 width 时 python-docx 会用图片自带 dpi 推导 height"）与 python-docx 1.2.0 实际行为不符——`scaled_dimensions` 只给 width 时按像素纵横比推 height，不经 DPI；01 的说法才是对的，但 02 的高度上限/分辨率警告仍需要像素数据。
- **修复方向**：补一条 R13 裁决。建议：允许"自研极小 PNG IHDR 读取（struct 解析前 33 字节，仍零依赖、绝不读 pHYs）"，保留 02 的高度上限与分辨率警告；修订 01 §8.2 措辞为"不读 DPI 元数据（pHYs），允许读 IHDR 像素宽高"；height 是否双写按 python-docx 实际行为定为"只传 width（触发高度上限时才双传）"。
- **【已闭环：R13】**

### HIGH-2【矛盾残留】R4 吸收规则与 02 PB-B 补插规则合并后不完备：摘要/第一章边界会被二次补插
- **涉及**：05 R4 vs 02 §D.4 PB-B、01 §5.1
- **描述**：R4 裁决"H2 边界与分节边界重合时，HrToken 被 breaks.py 吸收，不生成 PageBreakIR"。但 02 PB-B 原文是"**每个 H2 块（三类均含）前若无紧邻 PageBreak → 补插**"。裁决后摘要 H2（行13）与第一章 H2（行35）前不再有 PageBreakIR（被分节符接管），PB-B 按原文扫描会认为"缺分页"而**补插回去**——分节符换页 + PageBreakIR 双重换页 = 空白页，恰好重演历史问题3。PB-C 去重救不了场（它只合并相邻 PageBreakIR，管不到"分节符+分页符"异构叠加）。R4 只修订了"消费侧"（吸收 HrToken），没有同步修订"生产侧"。
- **触发场景**：实现者严格按 02 §D.4 字面实现 PB-B → PageBreakIR=13 而非 11 → 门3 断言 3+13=16≠14 → Fatal，产物无法产出（幸而断言能拦住，不会静默流出，但每次转换必失败）。
- **修复方向**：把 PB-B 改写为"每个 H2 块前若无紧邻 PageBreak **且该边界未被 SectionPlan 分节符接管** → 补插"；同时明确 PB-C 的去重范围声明不含分节符（由 PB-B 前置条件保证不叠加）。
- **【已闭环：R14】**

### HIGH-3【流程缺陷】门3 分页计数断言未随 `--no-appendix-page-break` 参数化：合法配置必然 Fatal
- **涉及**：05 R4 公式、03 §3.3 第9项（Fatal 级）、04 §1.1 `--no-appendix-page-break`
- **描述**：R4 修正公式"分节符(3)+PageBreakIR(11)=H2数(13)+目录合成节(1)"是按默认配置推演的常数式。04 提供了 `--no-appendix-page-break` 开关（架构 N3 明确保留），开启后附录B/C/D 前不补插，PageBreakIR=8，3+8=11≠14。门3 第9项 A 是 **Fatal 级断言**。没有任何文档定义断言公式如何随开关取值变化。
- **触发场景**：用户执行文档示例中明确给出的 `python -m md2docx report.md --no-appendix-page-break`（04 §1.5 原文示例）→ 门3 断言失败 → 删除临时文件、exit 2，合法配置 100% 转换失败。
- **修复方向**：断言公式参数化："分节符数 + PageBreakIR 数 = 未被分节符接管的应分页 H2 边界数 + 分节边界数"，从 IR 侧同一来源（breaks.py 的规划结果）取数——即门3 应校验"渲染产物与 SectionPlan/PageBreakIR 规划一致"，而非校验一个写死的算式（写死算式本身就是 F.1 V7"写死的数量断言"的近亲）。
- **【已闭环：R15】**

### HIGH-4【接口硬冲突】CLI `--version` 参数重名：元数据字段与 argparse 标准动作冲突
- **涉及**：04 §1.1 可选参数表
- **描述**：参数表同时定义了 `--version`（str，元数据补充值）与 `--version`（flag，打印转换器版本号）。同一 parser 注册两次 `--version`，argparse 直接抛 `ArgumentError`，CLI 无法启动。
- **触发场景**：任何一次 CLI 调用。
- **修复方向**：元数据参数改名为 `--doc-version`，转换器版本保留 `--version`。同时提醒：R12 已裁决采纳的 `--build-date` 尚未补入 04 参数表。
- **【已闭环：R16】**
