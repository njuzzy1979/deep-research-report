## 0. 总纲：算法在管道中的位置与全局原则

### 0.1 处理阶段与算法归属

```
[阶段P] 预处理（字符层/行层，纯文本）
   P0 二进制归一（R-01/R-02）
   P1 行级/行内清理正则（R-03 ~ R-09，代码上下文保护）
        ↓ 干净的 UTF-8 / LF 文本
[阶段S] 解析（文本 → 块级 IR：段落/标题/表格/图片/HR/列表）
[阶段A] IR 语义 pass（本文档核心算法全部在此层）
   A1 封面元数据提取（§D.5）
   A2 图片映射解析 + 宽度计算（§A）
   A3 表格题注三件套关联（§B）
   A4 标题语义分类 + 编号剥离重编（§C）
   A5 分页语义归一（§D.4，与架构设计师接口）
   A6 交叉引用一致性校验（§E，只读校验，不改 IR）
        ↓ 语义完备的 IR + 警告/错误清单
[阶段R] 渲染（IR → docx，本文档不覆盖）
[阶段V] 转换报告输出（§E.4 内容契约）
```

**关键分层纪律（历史问题8的根治）**：一切"删除/改写文本"的操作只发生在阶段P；一切"需要上下文/结构判断"的操作只发生在阶段A（在块级 IR 上做，且题注/元数据是**提取转移**而非文本删除）；阶段R 零文本清理。历史上"清理逻辑删掉了刚嵌入的题注"（问题8），根因是清理与嵌入在同一文本层反复互踩——本设计中题注一旦被 A3 关联即成为 IR 对象字段，任何文本正则都无法再触及它，该问题在架构层面不可复现。

## A. 图片映射动态解析算法（最高优先级）

### A.1 目标与输入输出

- **目标**：从 `![alt](path)` 语法一步解析出（图号 X-Y，题注全文，绝对文件路径，强制嵌入宽高）四元组。代码中**零**图号→文件名→图注映射（根除 FIGURE_MAP 缺陷，D1）。
- **输入**：图片 IR 块（解析器已从整行 `![...](...)mai` 语法生成，携带 alt 原文、path 原文、源行号）；md 文件所在目录；CLI 可选参数 `--figures-dir`。
- **输出**：`ImageIR{ fig_no:(X,Y)|None, caption:str|None, abs_path:str, emb_w_cm:float, emb_h_cm:float, warnings:[] }`

### A.2 图片行识别正则（解析层约定）

```
RE_IMG_LINE = ^\s*!\[([^\]]*)\]\((.+)\)\s*$
```

- 整行图片才构造块级 ImageIR。alt 组用 `[^\]]*`（alt 内的半角括号如 `(2009-2026)` 不影响，因 alt 以 `]` 定界）；path 组用贪婪 `(.+)` 配合行尾锚定，天然取到**行内最后一个** `)`，路径内含括号也安全。
- path 提取后二次处理：① 若匹配 `^(.*?)\s+"[^"]*"$`（md 可选 title 语法）则丢弃 title 部分；② 去首尾空白。
- **行内混排图片**（图片语法与其他文字同行，实测 0 处）：抽出图片为独立块置于该段之后渲染，剩余文本按普通段落处理，报 W-IMG-07。不静默丢弃。

### A.3 图号/题注解析正则（核心）

```
RE_FIG_ALT = ^图(\d{1,2})-(\d{1,2})(?:[ 　]+|[：:][ 　]*)(.+)$
```

设计要点（逐条对应实测语法事实）：

1. **分隔符是"紧跟图号的空格或冒号"，不是"首个冒号"**。实测样本分隔符全部为半角空格（9/9），但历史文档曾用冒号（pitfalls 问题8 的 `- 图2-1：...`），故兼容 `[ 　]+`（半角/全角空格）与 `[：:]`（全角/半角冒号）两种，且冒号仅在**紧贴图号**时才是分隔符。
2. **题注内部冒号不参与切分**：正则先锚定 `图\d-\d`，随后第一个字符才判定分隔符，题注余下部分整体捕获——`图1-1 研究框架总览：SSA能力闭环与竞争阵营映射`（行65）解析为 X=1, Y=1, 题注=`研究框架总览：SSA能力闭环与竞争阵营映射`，内部冒号完整保留。
3. **括号全半角均无影响**：括号只出现在题注捕获组内部（如行105 `(2009-2026)` 半角、其他图注含全角括号），`(.+)` 整体捕获。
4. **归一化输出题注** = `图{X}-{Y} {题注}`（分隔符统一为一个半角空格），与 V3.0 §4.2 图注格式（`图3-2 2025-2030年全球OOS市场规模预测…`）一致。渲染位置图下方居中（渲染器职责）。
5. alt 整体不匹配 RE_FIG_ALT → 该图为"无图号普通图片"，见 A.6 决策表。

### A.4 路径解析算法

```
resolve_image_path(raw_path, md_dir, cli_figures_dir):
    if raw_path 匹配 ^https?://  → 返回 URL 标记（W-IMG-08，不嵌入，
        渲染为文字占位段“【网络图片：{alt}（{url}）】”）
    候选列表（按序尝试 os.path.isfile，取第一个命中）:
      1. raw_path 为绝对路径 → 直接用
      2. normpath(join(md_dir, raw_path))            # 相对路径以 md 文件目录为基准（主路径）
      3. 若 cli_figures_dir 非空:
           normpath(join(cli_figures_dir, basename(raw_path)))   # CLI 覆盖：仅按文件名查
      4. 对 2/3 各追加一次 urllib.parse.unquote 解码后的变体      # 防 URL 编码中文路径
    若命中路径扩展名为 .svg:
      在同目录找同名 .png → 有则替换使用并记 I-CLN-05（SVG→PNG 替代）
                          → 无则 E-IMG-02（D2：SVG 嵌入实测不可行）
    全部候选失败 → E-IMG-01（默认中止；--allow-missing-figures 时降级
        为 WARN + 渲染文字占位段“【缺失图片：图X-Y 题注（原路径）】”）
```

主基准是 md 文件目录（实测 `research/drafts/final-report.md` 中 `../figures/` 相对该目录 9/9 可解析）。CLI 覆盖目录是**追加候选**而非替换，保证默认零参数即可正确转换。

### A.5 强制宽度算法（新发现高优坑：6/9 PNG 无 pHYs）

```
IMG_MAX_W_CM = 14.0    # V3.0 §5.2 单栏上限，可配置
IMG_MAX_H_CM = 20.0    # 防超高溢出（A4 版心高约 24.6cm，留题注与段距余量）

compute_embed_size(png_path):
    (px_w, px_h) = 读取 PNG IHDR 像素尺寸        # 仅像素宽高。
    # ！！绝不读取 pHYs/DPI：6/9 样本图无 pHYs，3/9 仅 200dpi，任何 DPI
    # 推导路径都会产出 60-90cm 巨幅或错误尺寸（data_full.md §2.2）
    aspect = px_h / px_w
    w = IMG_MAX_W_CM; h = w * aspect
    if h > IMG_MAX_H_CM: h = IMG_MAX_H_CM; w = h / aspect
    嵌入时 width 与 height **双参数显式传入**
    # 只传 width 时 python-docx 会用图片自带 horz/vert dpi 推导 height，
    # pHYs 缺失或横纵不等时纵横比可能失真——双写彻底切断 DPI 依赖链
    分辨率提示: px_w < 1102 → W-IMG-02（<200dpi@14cm）
               1102 ≤ px_w < 1654 → I-IMG-03（不足300dpi，仅信息）
    return (w, h)
```

统一 14cm 而非"取原图与 14cm 的较小值"：本管道图片均为整幅图表（实测 1779-2600px），不存在小图标场景；低分辨率经 W-IMG-02 暴露而非缩小回避。

## B. 表格题注动态解析算法

### B.1 目标与输入输出

- **目标**：`**表X-Y 标题**` 加粗行 → 表格 → `*数据来源：…*` 斜体行三件套在 IR 层关联为一个 TableIR；无题注表按无题注正常渲染。表号/表注 100% 取自文档（D1）。
- **输入**：块级 IR 序列（空行在块化时已消失，块间距离即逻辑相邻）。
- **输出**：`TableIR{ tbl_no:(X,Y)|None, caption:str|None, source_line:str|None, rows, warnings }`；被关联的题注块与来源块**从正文流移除**（提取转移，非删除）。

### B.2 关联算法（扫描窗口的精确定义）

```
RE_TBL_CAPTION = ^\*\*表(\d{1,2})-(\d{1,2})(?:[ 　]+|[：:][ 　]*)(.+?)\*\*$
                 # 施加于段落块全文（trim 后）。整段必须恰好是一个加粗题注，
                 # 无任何其他文字。分隔符规则与 RE_FIG_ALT 相同（空格/紧贴冒号兼容）。
RE_TBL_SOURCE  = ^\*([^*].*)\*$  且捕获组内容匹配 ^(?:数据)?来源[：:]
                 # 整段单星斜体 + “来源”前缀双条件。内容含《》（）等标点均安全；
                 # 内容若含星号则不识别（按普通段落渲染，无害降级，记录为已知限制）。

associate_tables(blocks):
    for 每个 TableBlock T (索引 i):
        # 向前窗口 = 严格紧邻前一块（距离恒为 1；实测题注与表格间仅隔空行，块化后即相邻）
        P = blocks[i-1]
        if P 是段落块 且 P.全文匹配 RE_TBL_CAPTION:
            T.tbl_no=(X,Y); T.caption=归一化“表X-Y 标题”; 移除 P（转移入 T）
        else:
            T 为无题注表（tbl_no=None, caption=None）——不猜测、不匹配、
            不套用任何关键词列表，按 V3.0 表格样式正常渲染，仅无题注行
        # 向后窗口 = 严格紧邻后一块
        S = blocks[i+1]
        if S 是段落块 且 S 匹配 RE_TBL_SOURCE:
            T.source_line = 捕获内容; 移除 S（转移入 T）
    # 孤立题注检测（关联完成后全表扫描）：
    for 每个仍在正文流中的段落块 Q:
        if Q.全文匹配 RE_TBL_CAPTION:
            W-TBL-01（保留按普通加粗段渲染，不删除——它可能是表格丢失的信号，
            删除会掩盖问题；渲染出来人工可见）
```

窗口定义依据实测：三张正文表（表2-1/4-1/6-1）结构 100% 一致——题注行、隔 1 空行、表格、隔 1 空行、来源行（data_full §1.5）。块化后空行消失，"紧邻块"即唯一正确窗口；**不做多块回溯扫描**（回溯窗口会把碰巧出现在表格前几段的加粗文字误认成题注，是误关联的根源）。

## C. 标题语义分类与编号算法

### C.1 层级映射约定（消除 md 级与 Word 级命名歧义）

| md 级 | Word 样式级 | 语义 | 编号 |
|---|---|---|---|
| H1（仅1个） | 封面标题 | 报告主标题 | 无 |
| H2 | Heading 1 | 章 / 前后置件 / 附录（三分类） | 章：第X章（中文数字）；前后置件：无；附录：附录A/B/… |
| H3 | Heading 2 | 节 | X.Y |
| H4 | Heading 3 | 小节 | X.Y.Z |
| H5/H6 | Heading 4/5 | 段落小标题 | 无（V3.0 §3.2 H4/H5 不编号） |

多个 H1：首个为主标题，其余降级按章处理 + W-HDR-03。

### C.2 编号剥离正则集（兼容式：带/不带编号两种输入均正确）

所有剥离动作逐条记 I-CLN-05（行号+剥离内容+剥离后标题），误伤可审计。

| ID | 正则 | 作用对象 | 误伤防护 |
|---|---|---|---|
| N-01 | `^第\s*([一二三四五六七八九十百零两]{1,4})\s*章\s*[：:、．.]?\s*` | H2 章（中文数字） | "第"字+"章"字双锚定，正文标题以"第X章"开头且非编号的概率趋零 |
| N-02 | `^第\s*(\d{1,3})\s*章\s*[：:、．.]?\s*` | H2 章（阿拉伯） | 同上 |
| N-03 | `^(\d{1,2})[.．](\d{1,2})[.．](\d{1,2})[.、]?(?:[ 　]+\|(?=[一-鿿（【《"「]))` | H4 小节 | 编号后必须跟空格或 CJK/全角开引（lookahead 不消耗）——`1.2.3beta` 类版本号标题不命中 |
| N-04 | `^(\d{1,2})[.．](\d{1,2})[.、]?(?:[ 　]+\|(?=[一-鿿（【《"「]))` | H3 节 | 同上：`3.5G技术` 后跟拉丁字母不命中、保留；`2.5万亿元市场` 类极端标题会被误剥（万为CJK），靠 I-CLN-05 台账+连续性校验暴露，记录为已知残余风险 |
| N-05 | `^(\d{1,3})[.、．]\s*` | H2 章（`3、技术架构` 型） | 要求数字后紧跟点/顿号——`2026年展望` 不命中 |
| N-06 | `^([一二三四五六七八九十]{1,3})[、．.]\s*` | H2 章（`一、导论` 型） | 同上 |
| N-07 | `^附录\s*([A-Za-z]\|[一二三四五六七八九十]{1,3})?\s*[：:、．.]?\s*` | H2 附录 | "附录"双字锚定 |

md H5/H6 **不剥离**任何编号（零误伤优先，实测 0 处 H4+）；若命中 N-03~N-06 形状仅报 W-HDR-04 提示人工。

### C.3 H2 三分类状态机（判定顺序即优先级）

```
FRONT_BACK_WORDS = {摘要, 执行摘要, 目录, 图表目录, 参考文献, 索引, 术语表,
                    前言, 导论, 绪论, 引言}   # 后 4 词为 V2.1 扩展，见 §C.3.1 R-FM
   # 词表来源：V3.0 §一 文档结构表（结构件名，属允许字面量，见 §F）

classify_h2(title_raw):
    视图列表 = [title_raw.trim()] + [依次试用 N-05/N-06 剥离后的文本]
    # 第1优先：前后置件——任一视图与词表精确匹配（允许尾随全/半角冒号）
    if ∃v ∈ 视图列表: v 匹配 ^(摘要|执行摘要|目录|图表目录|参考文献|索引|术语表)[：:]?$
        → 类别=FRONT_BACK；不编号；不占章计数；
          特例：“目录” → 内容由 TOC 域替代，其下的手动目录行全部丢弃并记 I-CLN-05
    # 第2优先：附录——N-07 前缀命中
    elif title_raw 匹配 N-07:
        → 类别=APPENDIX；剥离出原字母（可空）与标题文本；
          重编字母 = 按 APPENDIX 出现顺序取 A,B,C,…（>26 报警告）；
          原字母存在且 ≠ 重编字母 → W-HDR-02；渲染为“附录A：标题”（全角冒号）
    # 默认：章——无论是否带编号
    else:
        → 类别=CHAPTER；chapter_index += 1；
          依次尝试 N-01/N-02/N-05/N-06 剥离，命中则记录原编号（中文数字经 cn_to_int 转换）；
          渲染编号 = “第” + int_to_cn(chapter_index) + “章”，编号与标题间空 1 格（V3.0 §4.1）
```

分类为 FRONT_BACK 的区段内若出现 H3+ → 不编号渲染 + W-HDR-05。APPENDIX 区段内 H3 → 不编号（默认；可配置 `appendix_subsection_numbering` 生成 A.1 形式，默认关，因样本无实例、V3.0 无规定）。

### C.3.1 前置件区（FRONT_MATTER）识别机制（R-FM，V2.1 正式化）

**背景与动因**：`FRONT_BACK_WORDS` 词表原始 7 词（摘要/执行摘要/目录/图表目录/参考文献/索引/术语表）仅覆盖"单个 H2 即一个前置件"的情形。但真实报告常见"前言/导论"作为**一整章 H1**，其下辖多个无编号 H2（问题提出、研究目标、研究方法概述、报告结构说明……）。这些 H2 若走 §C.3 默认分支会被编为"第一章/第二章…"，挤占正文章序（历史缺陷 P0-1）。为此 V2.1 引入 `HeadingKind.FRONT_MATTER` 与本节的区间识别机制，作为 §C.3 三分类之外的**前置件区**补充规则，正式纳入设计规格（此前仅存在于实现代码，属未经设计审查的野生扩展）。

**词表扩展**：`FRONT_BACK_WORDS` 增补 4 词——前言 / 导论 / 绪论 / 引言。出处：V3.0 §一文档结构表将"前言/导论"列为标准前置结构件；判别口诀（§F.2）成立——"换一份完全不同主题的报告，'前言'二字是否仍必须出现在代码里？是 → 结构关键词，允许"。

**R-FM 算法（区间驱动，非计数驱动）**：

```
前置件区 = [起点, 终点)  在标题 Token 文档序上的一段连续区间
起点：首个 H1 的文本经 is_front_back() 判为前置件词
      （含复合形式，见下）→ 置 in_front_matter=True
终点（遇任一即退出，当前 H2 不再归前置件，改按 §C.3 正常分类）：
      (a) H2 携带显式章编号 —— N-01/N-02/N-05/N-06 任一命中（"第X章"/"3、"/"一、"）
          ——显式编号是"正文开始"的可靠结构信号；
      (b) H2 命中附录前缀 N-07 —— 附录属后置件，走 APPENDIX 分支。
区间内标题的分类：
      · H2 未命中终止条件 → FRONT_MATTER（number=None, display_number=""，不占章计数）
      · H2 命中 FRONT_BACK_WORDS（如区内再现"目录"）→ 仍按 §C.3 第1优先归 FRONT_BACK
      · H3 → FRONT_MATTER（剥离其数字前缀仅为清洁，不赋编号）
```

**为何用"结构信号边界"而非"数量上限"**：早期实现用魔法常量 `FRONT_MATTER_MAX_H2=6` 截断前置件区——这既是硬编码红线气味（§F.1：写死数量断言 V7），又无法适配前言 H2 数量不同的报告（少则漏判、多则误吞正文）。R-FM 改由文档自身的结构信号（显式章编号 / 附录前缀）划定边界，对任意前言 H2 数量都成立，且"换报告零改码"。

**is_front_back() 复合词判定（两级，后级仅在前级失败时启用）**：

```
is_front_back(text):
    clean = text.strip().rstrip("：:").strip()
    # 第1级：整体精确匹配（与 7 词时代逐字等价，纯净标题零行为漂移）
    if clean ∈ FRONT_BACK_WORDS: return True
    # 第2级：复合前置件标题——按 [/ 、 ， 空白] 切分，要求每一非空段都精确命中
    segs = [s for s in split(clean, r"[/、，,\s]+") if s]
    if len(segs) >= 2 and (∀ s ∈ segs: s ∈ FRONT_BACK_WORDS): return True
    return False
```

**误伤边界论证（关键，回应"包含匹配是否误伤正文标题"）**：判定采用**"分段后全部段整词相等"**，而非"子串包含"或"任一段命中"，据此排除三类误判：

| 输入标题 | 切分/匹配 | 判定 | 说明 |
|---|---|---|---|
| `前言/导论` | ["前言","导论"] 全命中 | **True** | 目标场景，正确识别 |
| `绪论、引言` | ["绪论","引言"] 全命中 | **True** | 顿号复合，正确识别 |
| `研究方法目录结构` | 无分隔符、整串不在词表 | **False** | 含"目录"二字**子串**但不误判（拒绝子串包含语义） |
| `背景/导论对比` | ["背景","导论对比"] 有段未命中 | **False** | 复合中含"导论"但非整段相等，不误判（拒绝"任一段命中"语义） |
| `导论`（裸 H2） | 整词命中（第1级） | **True→FRONT_BACK** | 见下方"裸前置件词 H2"裁决 |

**裁决：裸前置件词作 H2 的归类**。`## 导论`（无编号、H1 非前置件）在 7 词扩至 11 词后归 FRONT_BACK（不编号），**不再**如 §C.6 早期推演那样编为"第一章"。理由：一个标题字面就叫"导论/前言/绪论/引言"的章，语义上就是前置件，赋予"第一章"反而错误；作者若确要将其作正文首章，应写"第一章 导论"（N-01 命中后剥离编号、归 CHAPTER）。此裁决使 §C.6"假想输入 `## 导论` → CHAPTER"一行作废，以本节为准。

**索引一致性要求（P0-1 根因B 的设计约束）**：`FRONT_MATTER` 同时可能来自 H2 与 H3（一个 kind 对应两个 md 级）。因此**任何"按 kind 反推 md level"的索引都必须显式登记 FRONT_MATTER，或改用 source_line 单键索引**。builder 的 heading 索引采用后者（source_line 全局唯一），从索引层根除"新增 kind 忘记登记 level→查找返回 None→标题静默丢失"的失配。渲染层 `render/headings.py` 的 `_KIND_TO_LEVEL` 须显式登记 `FRONT_MATTER→Heading 2`（与 ABSTRACT 同级，均为无编号前置件），不得走 `.get(kind, 4)` 兜底。

**builder 步骤6 的 kind 敏感跳过规则（P006-2，补充索引一致性）**：H1 token 是否进入 `elements` 由**重编后的 kind** 决定，**不由 md level 决定**——当且仅当 `hir.kind == HeadingKind.MAIN_TITLE` 时跳过（主标题由 cover.py 单独渲染）；被重编为 CHAPTER 或 FRONT_MATTER 的 H1 照常入 elements 渲染。原实现 `if t.level == 1: continue` 按 md level 一刀切，导致：(1) 下述 R-FM 起点扩展归为 FRONT_MATTER 的前言 H1 被静默丢失；(2) W-HDR-03 降级为 CHAPTER 的多余 H1 标题文本静默丢失且章序 +1（既有缺陷，本次一并闭环）。

**R-FM 起点扩展（P006-1：标题 + 前言双 H1 结构）**：原起点判定仅在"**首个** H1 命中前置件词"时激活前置件区。但真实报告常见结构为"`# 报告标题`（首个 H1）+ `# 前言/导论`（第二个 H1）"——此时前言 H1 是**第二个** H1，走多余 H1 分支被无条件降级为 CHAPTER（W-HDR-03），前置件区从不激活，其下辖前言 H2 被误编号。P006-1 将起点放宽为"**首个 H1，或标题 H1 紧邻其后、在遇到任何 H2 之前出现的第二个前置词 H1**"，用一次性窗口精确约束：

```
title_h1_pending：首个 H1 非前置词（是报告标题）时置 True——开启一次性窗口
body_started：处理到任何 H2~H6 即置 True——正文/前置件内容层已开始
起点扩展分支：若 title_h1_pending and (not body_started) and is_front_back(第二个H1)
    → 该前言 H1 归 FRONT_MATTER（渲染 Heading 2，不占章序，见"降级 vs 前置"裁决）
    → 不发 W-HDR-03（它不是重复正文章，是被识别的前置件起点），改发 I-HDR-06 留痕
    → in_front_matter=True，title_h1_pending 窗口一次性消费
否则（非前置词 / 正文已开始 / 窗口已消费）→ 维持 W-HDR-03 降级 CHAPTER
```

一次性窗口确保只有"标题 H1 紧邻前言 H1"这一结构能激活：第三个及以后的 H1、或中间已插入 H2 后再现的 H1，都走常规 W-HDR-03（回归保护见 T-P006-5）。

**"降级 vs 前置"裁决（P006-1）**：识别为前置件起点的前言 H1（如`前言/导论`）**归 FRONT_MATTER**（候选 a），不新增 kind、不保留 CHAPTER。理由：它语义上是前置件区入口标题，与下辖前言 H2 同属前置件、同为无编号；`render/headings.py:_KIND_TO_LEVEL` 已登记 FRONT_MATTER→Heading 2，渲染路径现成；FRONT_MATTER 不占 `chapter_index`，天然避免章序污染。代价——它作为 md H1 却渲染为 Heading 2 级，但合理：报告主标题（真正 H1）已由 cover.py 单独渲染，正文流无需第二个 Heading 1 级大标题。此裁决**依赖 P006-2**（否则前言 H1 被 `level==1` 跳过而丢失），二者强绑定同批落地。

**R-FM 终止条件（P006-3，候选 C1 裁决记录）**：终止信号**维持"显式章编号 / 附录前缀"两项不变**，本轮**不**扩展为启发式自动分层。对"无编号正文首章被前置件区误吞"（根因 C：如 `## 军事需求与现状分析` 不带编号无法触发终止），采**只告警不猜测**——前置件区内累计 N 个无编号 H2 时收尾发 `W-FM-01`，提示作者为正文首章补显式编号以标示正文起点。信息论上"无编号正文章"与"又一个前言 H2"不可判定，唯一诚实处置是告警而非猜测，零误伤、与 R-FM"换报告零改码"哲学一致。**已知限制**：官方 `front-matter.md` 夹具原正文章全无编号，故本次夹具修订为其正文章补显式编号（`第一章`/`第二章`），使夹具真正演示"前置件 + 正文"正确分层。候选 C2（启发式分层）经用户裁定（U-1）不在本轮采用，列为独立后续立项（见 07-p3-fixes-design.md §7 U-1）。

**新增 Issue 语义**：FRONT_MATTER 的产生不单独记 Issue（无编号、无剥离动作）；其内 H3 若被 §D 剥离数字前缀仍记 I-CLN-05（清洁台账）。起点扩展的前言 H1 记 I-HDR-06；前置件区无编号 H2 累计记 W-FM-01（见 §0.3）。


H3/H4 编号：分别对所在章维护节计数器/所在节维护小节计数器，渲染 `“{章序}.{节序}”` / `“{章序}.{节序}.{小节序}”`（阿拉伯数字，V3.0 §4.1）。剥离得到的原编号仅用于校验。

### C.4 阿拉伯数字↔中文数字转换算法

```
CN = "零一二三四五六七八九"
int_to_cn(n):                     # 1..999，覆盖章数上限绰绰有余
    if n < 10:  return CN[n]
    if n < 20:  return "十" + (CN[n%10] if n%10 else "")        # 11→十一, 10→十
    if n < 100: return CN[n//10] + "十" + (CN[n%10] if n%10 else "")  # 21→二十一, 30→三十
    s = CN[n//100] + "百"; r = n % 100                          # 百位处理
    if r == 0:  return s                                        # 100→一百
    if r < 10:  return s + "零" + CN[r]                         # 101→一百零一
    return s + CN[r//10] + "十" + (CN[r%10] if r%10 else "")    # 110→一百一十, 119→一百一十九
    # 注：r∈[10,20) 时 r//10==1 自然产出“一十”，符合“一百一十”规范写法

cn_to_int(s):                     # 剥离校验用；支持 零一..九 十 百 两(→2)
    纯数字 → int(s)
    否则按位值法：遇“百”当前数×100 累加，遇“十”当前数(空则1)×10 累加，末位数字直接加
    解析失败 → 返回 None（该标题按无编号处理，不报错）
```

### D.4 分页语义规则（与架构设计师的接口点，非清理正则）

历史 `---` 在预处理被"先删再插"（pitfalls 管道图），本设计改为**语义保留**：

- PB-A：解析层 `---` → PageBreak IR 节点（实测 11 处语义 100% 为 H2 边界分页，data_full §1.6）。
- PB-B（补插，D5）：IR 层扫描，每个 H2 块（三类均含）前若无紧邻 PageBreak → 补插 + W-PB-01。附录A/B/C/D 之间实测无 `---`，本规则使其各自另起一页（V3.0 §一"每一章从新的一页开始"的延伸；样本将触发 3 次补插：附录B/C/D 前）。
- PB-C（去重）：相邻多个 PageBreak 合并为一 + I-PB-02（根治问题3 的多源分页叠加）。
- PB-D（尾页）：文档末尾 PageBreak 之后若无实质内容块 → 移除该 PageBreak（V3.0 检查清单"无空白页"）。样本行521 的 `---` 之后有行523 斜体段（W-CLN-03 保留），故该分页保留。
- PB-E（缺口清理，P-007）：SectionPlan 定位首个 consuming 节（四节=首个 ABSTRACT H2；无摘要三节=首个 CHAPTER H2）后，移除位于其 `start_element_index` 之前的**全部 PageBreakIR**，每移除一个记 I-PB-03。动因：这些 PageBreakIR 落入渲染缺口（`_compute_section_ranges` 的首个 consuming 节 start 之前区间不属任何 range），既不换页也无留痕，而其换页需求已由 COVER→首个内容节的分节符（R4 NEW_PAGE）覆盖，属冗余分页。PB-E 使"规划的 PageBreakIR 数"与"渲染实际消费数"重新对齐——gate3 R15 期望值（`len(PageBreakIR in elements)`）自动恢复诚实，**无需改 gate3 计数逻辑**（G-10 单一事实源）。与 R4 分节符接管的关系：分节符已换页，前导显式分页冗余，PB-E 就地吸收。**职责边界**：PB-E 只清理 PageBreakIR；缺口内的**非分页前导元素**（普通段落、H1 后游离段、P-006 归为 FRONT_MATTER 的前言标题/段落）由渲染层 `_compute_section_ranges` 把首个 consuming 节起点扩展到 0 来归入首个内容节渲染（并入 ABSTRACT/BODY 节，四节方案落罗马页码/摘要页眉，见 §0.3 W-SEC-02 与 07-p3-fixes-design.md §3.3 / U-2），二者职责严格互补、不重叠。
- 渲染器只认 PageBreak 节点这一种分页来源（收敛单一触发机制，问题3/10/11 的架构性根治）。

### D.5 误伤防护总原则（每条规则的设计前提）

1. **整行锚定优先**：删行类规则（R-03/04/05/09强②/10）必须 `^…$` 整行匹配，正文段落中部的相似字样不可能命中。

3. **代码上下文保护**：fenced 与行内代码 span 内跳过一切规则（样本行478/490 的路径行内代码不受影响）。
4. **删除三分级**：确定性残留（特征极强）→ 自动删；疑似残留 → warn-and-keep；语义内容（元数据/题注）→ 提取转移。任何规则不允许"猜测性删除"。

## E. 交叉引用一致性校验（含"先见文后见图/表"检测，D11 + 任务G 合并）

### E.1 目标与输入输出

- **目标**：警告级只读校验（不阻断、不改写）：① 先见文后见图/表；② 0 引用；③ 引用不存在的编号；④ 位置性指代残留。
- **输入**：完成 A2/A3/A4 后的 IR（清理已完成——见 D.6 顺序效应）；**输出**：警告清单（W-REF-01~04）。

### E.2 扫描范围定义

```
实体表：figures[(X,Y)] = 图片块位置（块序+源行号）；tables[(X,Y)] = 表格块位置
引用扫描范围 = FRONT_BACK 与 CHAPTER 区段内的段落块与列表项块
排除：APPENDIX 区段全部内容（附录D图表索引是索引不是正文引用，计入会
      污染 0 引用检测）；表格单元格；已关联的题注/来源行（已移出正文流）；图片 alt
```

### E.3 检测算法

```
RE_REF   = (图|表)(\d{1,2})-(\d{1,2})          # 引用模式核心。“如图X-Y所示”“（图X-Y）”
                                                # “见表X-Y”“详见第四章图4-2”等全部句式
                                                # 都以此核心命中，无须枚举句式前缀
RE_POS   = (?<!以)[上下]图|(?<!以)[上下]表      # 位置性指代；(?<!以) 防“以下图表”误报
RE_ORPH  = ^图\d{1,2}-\d{1,2}(?:[：:]|[ 　]).{0,80}$   # 孤立图引用行形状（R-07 剥离列表
                                                # 前缀后的残留段；仅当该段落块前后均无图片块时）

check(ir):
    for 每个范围内段落块 P（按文档序）:
        for m in RE_REF.finditer(P.text): refs[(type, X, Y)].append(P.位置)
        for m in RE_POS.finditer(P.text): W-REF-04（行号+上下文±15字）
        if P.text 匹配 RE_ORPH 且 P 前后紧邻块均非图片块:
            W-REF-03 附注“疑似占位残留”（保留渲染）
    for (X,Y), pos in figures/tables:
        R = refs[(该类型, X, Y)]
        if R 为空          → W-REF-01 “图X-Y（行L）从未在正文中被引用”
        elif min(R) > pos  → W-REF-02 “先见图后见文：图X-Y位于行L0，首次正文引用在行L1”
        # min(R) < pos 即合规；图后再次出现“如图X-Y所示”不构成违规
    for refs 中不存在于实体表的 (X,Y) → W-REF-03 “正文引用了不存在的图X-Y（行L）”
```

### E.4 输出格式（转换报告内容契约）

每条校验项输出字段：`级别 | 代码 | 源行号 | 消息 | 原文摘录(≤60字)`。报告按 错误→警告→信息 分节，节内按源行号排序；末尾统计块：章/节/图/表数量、引用覆盖率（被引用实体数/实体总数）、清理动作数。呈现载体（stdout/伴生 .report.md）由接口设计师定稿，此处只约定内容契约。

### F.3 自动化检测手段（实现阶段落地规格）

1. **中文字面量扫描**：脚本 `check_no_hardcode.py`——提取 `src/` 全部字符串字面量中含 CJK 字符者（AST 遍历优于 grep，可跳过注释），与 `constants.py` 白名单集合做差；差集非空 → 检查失败并列出 文件:行号:字面量。白名单每项须带出处注释，脚本校验注释存在性。
2. **结构模式 grep**：`\bMAP\s*=\s*\{`、`CAPS\s*=\s*\[`、`MATCHERS`、键形如 `['"]\d+-\d+['"]` 的 dict 字面量 → 命中即人工复核。
3. **换样本金标准测试（终极判据）**：维护一份与真实样本题材完全不同的合成 md 夹具（不同标题/图表名/章数/含无编号标题变体），转换产物中的全部题注/编号/封面字段必须与夹具内容一致。**换报告零改码**通过 = 反硬编码达成；本测试列为 CI 必跑项。
4. **人工 Review 清单**：PR 描述中逐条勾选 V1-V9 不存在；新增任何中文字面量必须走白名单+出处流程。
