## R1【CONTRADICTION】图片文件缺失的默认分级（01 vs 02 vs 03 三方不一致）
- 01 §4.4：ERROR + 占位框渲染，继续，exit 1
- 02 §A.6：默认报错中止（E-IMG-01），`--allow-missing-figures` 降级警告
- 03 §3.1/§6：门1 Warning，跳过渲染不阻断
**裁决：采用 01 方案——ERROR 级：渲染占位框（框内显示原路径与图号）+ 继续转换 + exit code 1（非零）**。`--strict` 下升级为 FATAL 中止；`--allow-missing-figures` 降级为 WARNING（exit 0）。
理由：防"静默丢图"的本质要求是三重可见性（非零退出码 + 报告 ERROR 明细 + 文档内可见占位框），已全部满足；相比 02 的默认中止，保留"一次运行暴露全部问题"的诊断效率；03 的 Warning+exit 0 太弱，调度层无法感知，予以否决。02 的错误码 E-IMG-01 语义改为"ERROR（可降级/可升级）"；03 门1 该项分级同步升为 ERROR。

## R2【CONTRADICTION】exit code 语义（03 vs 01/04）
- 03 §12.1：0=成功 / 1=Fatal失败 / 2=成功但需人工关注
- 01 N9 与 04 §1.3：0=成功（可含WARNING）/ 1=完成但含ERROR / 2=FATAL或CLI参数错误
**裁决：采用 01/04 方案。** 理由：argparse 参数错误天然 exit 2（Unix 惯例），03 方案中 2 兼作"成功但需关注"会产生致命歧义；且 Unix 惯例数值越大问题越重。03 的"需人工复核"信号改由转换报告的"人工复核清单"小节承载（非空时控制台摘要显式提示），不占用退出码。

## R3【CONTRADICTION】分页机制的实现层（03 文本层 vs 01/02 IR 层）
- 03 §2.3.1：预处理阶段在文本层插/删 `---` 做分页规范化
- 01 N1 + 02 §D.4：分页收敛点上移为 IR 层 PageBreakIR（清理阶段不再插删 `---`）
**裁决：采用 01/02 的 IR 层方案（N1）。** 03 §2.3.1 的算法思想（逐 H2 边界检测已有锚点→补插→去重）完整保留，但执行位置迁移至 `assemble/breaks.py` 的 IR 层（消费 HrToken + HeadingIR 位置信息）。03 的"每个 H2 边界恰好一次分页"不变量与门3计数断言保留（按 R4 修正公式）。

## R4【GAP】分节符与分页符的重合去重规则（所有文档均未完整覆盖）
04 的四节方案（COVER/ABSTRACT/TOC/BODY）引入 3 个 NEW_PAGE 分节符，与 `---` 触发的 PageBreakIR 在部分 H2 边界重合（如行11 元数据→摘要、行33 摘要→第一章），若不去重会产生"分节换页+分页符"双重换页=空白页（重演历史问题3）。
**裁决（补位规则）：H2 边界与 SectionPlan 分节边界重合时，由分节符（NEW_PAGE 型）独占换页职责，对应位置的 HrToken 被 breaks.py 吸收（不生成 PageBreakIR，记 INFO）。**
样本推演：分节符 3 个（封面→摘要、摘要→目录、目录→正文第一章）；PageBreakIR 11 个（第二~八章前 7 + 附录A 前 1 + 附录B/C/D 自动补插 3）；行11/行33 的 `---` 被分节符吸收；行521 游离 `---` 按 R5 处理。
**门3 分页不变量修正为：NEW_PAGE 分节符数(3) + PageBreakIR 数(11) = H2 边界数(13) + 目录合成节(1)。**

## R5【CONTRADICTION】文档末尾游离 `---`（行521）的处置（02 vs 03）
- 02 PB-D：其后有实质内容（行523 斜体说明段）→ 分页保留
- 03 §2.3.1 步骤3：非 H2 边界的游离 `---` 一律删除
**裁决：采用 03 规则——游离 `---` 移除、不分页（INFO 留痕）；行523 文末斜体段保留渲染 + W-CLN-03 警告（02 R-11 保留）。** 理由：V3.0 §9.4"节间不使用分隔线"+§10.3"无空白页"；文末说明段独占一页无排版价值。02 的 PB-D 规则相应修订。
