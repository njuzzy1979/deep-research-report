- **上游依赖（触发前必须就绪）**：

- **下游消费者**：
  - 直接消费者：阶段 9.3"最终交付物核对"环节的执行人（可能是同一 Agent 会话，也可能是人工），消费转换报告 + docx 本身

### 1.2 幂等性要求

| 维度 | 要求 | 说明 |
|---|---|---|
| **语义幂等（强制）** | 同一输入 md + 同一 figures 目录，重复运行 N 次，产出 docx 的**结构化内容**（文本、图号、表格数据、样式引用、域代码结构）必须完全一致 | 可通过"用 python-docx 重新打开两次产物 → 提取纯文本+表格数据+图片数量 → diff"验证，纳入回归测试（见 §6） |
| **字节级幂等（不强制）** | 不要求两次运行产出的 .docx 文件字节完全相同 | python-docx/Word OOXML 会写入 `rsid`（修订会话 ID）等非确定性字段，字节级复现成本高、收益低 |
| **时间戳字段（显式控制）** | `core_properties.created`/`modified`/`last_modified_by` 三个字段必须由转换器显式赋值，不使用 python-docx 默认的"当前系统时间" | 默认行为会导致"仅仅因为运行时刻不同"就产生噪音 diff，干扰回归比对；建议固定为报告 `metadata.version` 中内嵌的日期值（如"2026年7月"→归一化为该月第1日），或允许 CLI `--build-date` 显式传入（参数命名由 InterfaceDesigner 定） |
| **中间产物落盘（清理后 md）** | **默认落盘并保留**，不作为纯内存中间态 | 位置：与目标 docx 同目录，命名 `<docx文件名(不含扩展名)>.cleaned.md`；默认保留供调试（沿用 SKILL.md 9.3 "清理后的 Markdown……调试用"的既有约定），可通过 CLI 开关关闭（开关命名由 InterfaceDesigner 定）；每次运行覆盖写入，不追加时间戳后缀（保持幂等：同一输入产出同一路径同一内容） |

### 2.2 各阶段前置条件 / 成功判据 / 失败行为

| 阶段 | 前置条件 | 成功判据 | 失败时行为 |
|---|---|---|---|
| 阶段0 前置检查 | CLI 参数已解析 | 输出目录可写、figures 目录存在（若 md 中有图片引用） | Fatal：输出路径不可写 → 中止，仅控制台报错（此时报告文件也可能无法落盘，退化为 stderr 输出） |
| 门1 输入校验 | 阶段0通过 | md 非空可读、UTF-8 可解码 | 空文件/不可解码 → Fatal 中止；图片缺失/格式不支持/题注格式异常 → Warning，继续 |
| 阶段1 预处理 | 门1通过（允许带 Warning） | 产出规整的 `\n` 分隔纯文本，分页标记已规范化 | 预处理本身不应抛出可恢复异常（正则清理是确定性操作）；若清理后文本为空 → 视为等价于"md为空"，Fatal |
| 阶段2 解析 | 阶段1产出就绪 | 生成完整 IR 对象树 | 解析器内部异常（非用户输入问题）→ Fatal，报告需注明"转换器内部错误"区别于用户输入问题 |
| IR 完整性自检 | 阶段2产出 | 每个图片/表格/标题节点的必需字段（path/caption/number 等）非 None | 关键字段缺失 → Fatal（说明解析器遇到未预期输入模式，需要修复代码而非用户重跑） |
| 门2 IR校验 | IR自检通过 | 所有校验项执行完毕并记录结果 | 全部降级为 Warning/Info/需人工复核，不阻断（严格遵循 D11"不阻断不改写"精神，扩展到整个门2） |
| 阶段3 渲染 | 门2完成 | docx 写入临时文件成功、python-docx 重新打开临时文件无异常 | python-docx 抛异常（样式KeyError等）→ Fatal，捕获并输出"渲染步骤+涉及元素"上下文，不产出临时文件之外的最终文件（避免半成品）|
| 门3 输出校验 | 渲染产出临时文件 | 13 项检查逐项执行完毕 | 密级复检命中 → Fatal（阻断产出，删除临时文件，防止带密级字样的文档流出）；其余项 Warning/Info，不阻断，临时文件原子重命名为最终产物 |
| 转换报告生成 | 门3完成（或任一环节Fatal中止） | 报告文件成功落盘 | 报告落盘失败（如输出目录中途被删除）→ 退化为仅控制台输出，不视为二次 Fatal（因主产物 docx 已生成或已明确失败，报告是附属产物）|

### 2.3 分页决策流程（D5 细化）

**核心不变量（必须双向证明成立）**：*每个 H2 边界恰好触发一次分页事件*，即"分页事件数 = H2 数量"（封面→第一个H2 也算一次边界，无特例）。

#### 2.3.1 分页标记规范化算法（预处理阶段执行，渲染阶段不再关心 H2 语义）

```
输入：清理后的、已确定编号剥离完毕的 Markdown 文本（按行）
输出：`---` 标记的位置集合，满足"每个 H2 标题前恰好一个`---`，其余位置无`---`"

步骤：
1. 扫描全文，定位所有 H2 标题所在行号，得到边界列表 H = [h1, h2, ..., hN]
   （本例 N=13：摘要 + 第一章..第八章 + 附录A..D）
2. 对每个 hi ∈ H：
   a. 向上回溯 hi 之前的空行区间，检测该区间内已存在的 `---` 数量 c
   b. 若 c == 1 → 该 `---` 保留，标记为 hi 的"分页锚点"
   c. 若 c == 0 → 在 hi 标题正上方插入一个 `---`（自动补插），标记为分页锚点
   d. 若 c >= 2 → 保留区间内第一个 `---` 作为锚点，其余全部删除（去重）
3. 扫描全文剩余的、不属于任何 H2 边界的游离 `---`（即不紧邻在某 H2 标题之前的 `---`，
   如样本行521"附录D结束→文末说明段"之间的`---`）：
   → 一律删除，不转换为分页事件；按 V3.0 §9.4"节间：不使用分隔线"处理，
     在预处理阶段的 Info 记录中登记"已移除非章节边界分隔线，原行号N"
4. 校验：处理完毕后，`---` 总数必须精确等于 len(H)（本例=13），
   若不等于 → 该算法自身实现缺陷，属于转换器内部错误（Fatal，不应在生产环境出现，
   仅用于单元测试自检）
```

#### 2.3.2 渲染层职责（保持极简，不重复判断 H2 语义）

- 渲染层遍历元素流时，**唯一**的分页触发规则：`isinstance(elem, HorizontalRule) → doc.add_page_break()`
- 封面结束后的分页单独处理（封面自身不是由 `---` 触发，是渲染流程固定的"封面→分节→第一内容页"逻辑，由 SystemArchitect/InterfaceDesigner 定具体实现）
- H1/H2/H3 段落本身**不触发**任何分页（无论是 `add_page_break()` 还是 `page_break_before=True`）——杜绝 tech_full.md 12点提到的"双重保险丝叠加导致空白页"
- `back_matter`（附录/文末）渲染函数**不含**任何独立的 `add_page_break()` 调用

#### 2.3.3 双向不可能性证明（历史问题3、11的规避）

- **问题11"首章未分页"不可能重现**：算法 2.3.1 对 H 列表中的每一个 hi 一视同仁处理（含 h1=摘要、h2=第一章），不存在"跳过第一个元素"的分支逻辑（旧代码 `first_chapter` 标志在新流程中没有对应物）。第一章作为 H 列表第2项，必然被步骤2处理并获得锚点。
- **问题3"分页符过多"不可能重现**：
  1. 每个 H2 边界的锚点数被算法强制去重为恰好1（步骤2c/2d）
  2. 非 H2 边界的游离 `---` 被算法删除而非转为分页（步骤3）
  3. 渲染层只有唯一触发点（§2.3.2），不存在 H1 段落触发 + `---` 触发 + back_matter 函数触发的三头并发架构（这正是旧问题3的根因）
- **回归校验网**：门3 增加"分页事件计数 == H2数量"的自动断言（见 §4 第9项），作为该不变量在生产环境的最后一道保险丝，即便预处理算法未来被修改出现回归，门3 也会捕获。

## 3. 校验门禁体系（三道门）

### 3.1 门1：输入校验（转换前）

| 检查项 | 检测方法 | 分级 | 处置 |
|---|---|---|---|
| md 文件存在性/可读性 | 二进制 `rb` 打开 | Fatal | 文件不存在/无读权限 → 中止 |
| md 编码可解码性 | 尝试 `bytes.decode('utf-8', errors='strict')` | Fatal | 解码失败 → 中止（无法确定文本内容，继续处理风险不可控） |
| md 非空 | 解码后文本 `strip()` 长度 > 0 | Fatal | 空文件 → 中止 |
| md 超大 | 文件字节数 > 阈值（建议 10MB，样本量级 90KB 的 100+ 倍，超出为异常拼接可能） | Warning | 不阻断，仅提示"请确认输入完整性" |
| BOM 检测 | 前 3 字节是否为 `EF BB BF` | Info | 自动剥离，不阻断 |
| 图片文件存在性预检 | 对全部 `![...](path)` 语法，以 md 所在目录解析相对路径，`os.path.isfile` 校验 | Warning | 缺失 → 该图降级为跳过渲染（原位置留占位提示文本），不阻断整体转换 |
| 图片格式预检 | 检测路径扩展名是否在支持白名单（PNG/JPEG/GIF/TIFF/BMP，取自 tech_full.md 第7点 `docx.image.SIGNATURES`） | Warning | `.svg` 或其他不支持格式 → 该图降级为跳过渲染，提示"请转换为 PNG 后重新引用"（呼应 D2） |
| 图片文件过大预检 | 单图文件字节数 > 阈值（建议 10MB） | Warning | 不阻断，仅提示"可能影响最终 docx 体积" |
| 表题注格式预检 | 扫描全部加粗行，检测是否匹配 `**表X-Y ...**`（X-Y 均为数字）模式（正则细节归 ModelDesigner） | Warning | 格式异常 → 不阻断，该行在解析阶段将被当作普通段落处理，对应表格按"无题注表格"降级渲染 |

**门1 整体策略**：仅"文件不可读 / 编码不可解码 / 文件为空"三类导致后续流程完全无法进行的情况 Fatal 阻断；其余全部 Warning，最大化产出可用文档。

### 3.2 门2：IR 校验（解析后，渲染前）

| 检查项 | 检测时机/方式 | 分级 | 处置 |
|---|---|---|---|
| 图表编号连续性 | 遍历 IR 中图/表节点，提取编号，检查章内序号从1连续无跳号（算法归ModelDesigner） | Warning | 不阻断，原样渲染，报告列出跳号位置 |
| 图表编号与所属章节匹配 | 图号章号部分与该图所在 H2 章号是否一致 | Warning | 不阻断，仅提示 |
| 交叉引用一致性 | 正文"（图X-Y）"预告文本 vs IR 中实际存在的图/表节点是否一一对应 | Warning | 有预告无图 / 有图无预告 → 均记录，不阻断 |
| 先见文后见图（D11） | 预告位置行号 vs 图/表实际位置行号先后顺序 | Warning | **不阻断、不改写**，严格遵循 D11 |
| 密级字样复扫 | 对 IR 全部文本节点执行关键词扫描（"绝密/机密/秘密/内部资料/内部参考/限内部使用"等） | **需人工复核**（独立于Warning/Error的第三类标记） | 自动过滤命中文本（按最小粒度删除关键词或整行，具体粒度归ModelDesigner），**同时**在转换报告"人工复核清单"高亮列出被过滤的原文与位置——因为自动删除敏感词存在误伤正常语境的风险（如"内部资料室"作为机构名的一部分），不能悄悄处理 |
| 红队过程标记残留复扫（D12） | 对 IR 全部文本节点扫描 `[红队 R\d+ 已...]`、`[已修正：...]` 模式 | Warning，且需标注"预处理清理规则可能存在遗漏" | 若门2 仍检出残留，说明预处理阶段的正则未覆盖到该变体，这是**转换器自身缺陷信号**（区别于用户输入问题），报告需明确区分二者 |
| 分页事件预校验 | 统计 IR 中"待渲染的分页事件"数量 | Info | 提前记录，供门3 最终核验用（见§4第9项），此处不阻断 |
| 附录B非结构化内容（D6） | 不适用 | — | 转换器按原文渲染，门2 不对其做结构化校验 |

**门2 整体策略**：全部降级处理，不阻断渲染（唯一的"需人工复核"标记也不阻断，但会在转换报告中以独立小节高亮，供用户在拿到 docx 后核对）。IR 完整性自检（见 §2.2 表）是门2 之前的独立步骤，属于"转换器健壮性"范畴，与门2"输入内容质量"范畴分开，只有它可以 Fatal。

### 3.3 门3：输出校验（docx 生成后）—— V3.0 §10.3 13 项映射表

方法论：门3 用 **python-docx 重新打开刚生成的临时 docx 文件**（而非复用内存中的 Document 对象），模拟"下游消费者拿到文件后再检查一次"的视角，避免"写入时看似正确但序列化/保存环节引入问题"被漏检。

| # | V3.0 §10.3 检查项 | 分类 | 校验方法 | 失败处置 |
|---|---|---|---|---|
| 1 | 封面完整（标题/副标题/机构/日期/版本） | **A 自动校验** | 重新打开 docx，读取首节前 N 段文本，与 IR 中提取的 title/subtitle/org/version/date 字符串做子串比对 | 缺失任一要素 → Warning，列出缺失字段 |
| 2 | 全篇无密级标注 | **A 自动校验（唯一可 Fatal 项）** | 遍历生成 docx 全部 paragraph.runs 提取纯文本，执行与门2相同的关键词扫描，命中数须为0 | 命中 → **Fatal**，删除临时文件，中止产出（这是强制红线条款，说明门1/门2的过滤失效，不能放行） |
| 3 | 目录自动生成且可跳转 | **混合**：结构=A，可跳转体验=B | A：检查 document.xml 中 TOC 域 fldChar 序列 begin→separate→end 齐全；B：需人工在 Word/WPS 中 F9 更新域后点击测试 | A失败→Fatal（域结构缺陷是转换器代码bug）；B 写入人工复核清单 |
| 4 | 所有章节编号连续无跳号 | **C 转换器保证**（+ A 兜底断言） | 编号由转换器生成（D3），逻辑上必然连续；门3 仍做断言"第X章"的X序列=1..N 连续，作为最后一道保险 | 断言失败 → Fatal（转换器渲染逻辑缺陷） |
| 5 | 所有图表编号连续且与正文交叉引用一致 | **A 自动校验** | 重新解析生成 docx 中题注文字，提取编号，复核连续性+交叉引用一致性（与门2逻辑相同，此处目的是确认渲染层未引入新的不一致） | 不连续/不一致 → Warning（沿用门2已记录的结果，若门3发现门2未记录的新问题 → 额外标记为"渲染引入的新问题"，视为转换器缺陷线索） |
| 6 | 表格全框线（含竖线），跨页表头重复 | **A 自动校验** | 遍历所有 table，检查 tblPr/tblBorders 六方向 XML 是否存在且 sz 值符合规范（顶底=12，内线=4）；检查每表首行 trPr 是否含 tblHeader | 缺失/值错 → Warning |
| 7 | 页码正确（摘要罗马数字，正文阿拉伯数字） | **混合**：域结构=A，实际渲染结果=B | A：检查各 section 的 sectPr/pgNumType 的 fmt/start 属性是否符合预期（罗马/阿拉伯，正文从1重新起算）；B：需 F9 更新域后翻页人工确认 | A失败→Warning；B写入人工复核清单 |
| 8 | 页眉页脚完整 | **混合**：结构=A，动态内容=B | A：检查页眉 pBdr 底线设置、页脚 PAGE 域结构是否存在；B：页眉简称文字+ STYLEREF 跟随当前章节标题的动态效果需人工翻页确认 | A失败→Warning；B写入人工复核清单 |
| 9 | 无空白页/多余分页符 | **混合**：分页计数=A，肉眼空白页=B | A：统计生成 docx 中 page_break 元素总数，断言等于"H2数量+封面1次"（见§2.3不变量）；B：翻页确认无异常空白页（尤其内容恰好占满一页导致的边界情况，程序难以100%判定） | A失败→**Fatal**（说明§2.3算法未生效，是回归） ；B写入人工复核清单 |
| 10 | 链接可点击 | **A 自动校验 / N-A** | 若 IR 检测到 `[text](url)` 语法，校验 document.xml.rels 中存在对应 hyperlink 关系且正文含 w:hyperlink 元素；若输入中无超链接（如本样本 final-report.md 实测0处）→ 该项标记为 **N/A（本次转换无超链接内容）**，不计入 PASS/FAIL | 有链接但结构缺失 → Warning |
| 11 | 字体嵌入（或使用系统通用字体） | **C 转换器保证 + A 自动校验** | 转换器只使用系统通用字体白名单（宋体/微软雅黑/Times New Roman/Consolas），不做真正的字体嵌入（embedTrueTypeFonts 超出 python-docx 能力，tech_full.md 未验证该能力）；门3 扫描生成文档全部 rFonts 设置，确认只出现白名单内字体名 | 出现白名单外字体名 → Warning |
| 12 | 文末参考文献完整 | **B 人工复核 / 已知限制** | 附录B在样本中不是结构化文献列表（D6：转换器按原文渲染，不做结构化提取）；此项**无法**由转换器自动判定"完整性"，在报告中显式标注"已知限制：转换器不生成结构化参考文献列表，原样渲染附录B原文，信源清单完整性需用户自行确认外部 CSV" | 始终标记"需人工复核 + 已知限制"，不计入 PASS/FAIL 二元判定 |
| 13 | 文件大小合理（<50MB） | **A 自动校验** | `os.path.getsize(output.docx) < 50*1024*1024` | 超限 → Warning（本项目量级约1-2MB，正常情况不会触发，超限说明图片资源异常） |

**汇总**：13 项中，A 类（可自动校验）7 项（1需拆分/4/6/9部分/11/13 + 2的唯一Fatal），A+B 混合 5 项（3/7/8/9/10部分），纯 C 类（转换器保证）1 项（4，仍配 A 兜底），B 类/已知限制 1 项（12）。**只有"密级复检"（第2项）和"分页事件计数"（第9项A部分）可以在门3 判定为 Fatal**，其余全部 Warning 或写入人工复核清单，不阻断产物交付。

### 3.4 三道门的输出如何汇入转换报告

```
门1 结果 ──┐
门2 结果 ──┼──→ 转换报告汇总器 ──→ 按分级分类到：错误/警告/信息/人工复核清单 四个桶
门3 结果 ──┘                        （见 §5.1 报告结构）
```

- 每道门产生的记录统一结构：`{行号/元素定位, 门禁编号, 检查项, 分级(Fatal/Warning/Info/需人工复核), 描述, 建议动作}`
- 报告汇总器按分级分桶，同时保留"来源门禁编号"字段，供调试时快速定位是哪道门发现的问题

### 5.1 报告结构

```markdown

| # | 检查项 | 结果 | 说明 |
```

### 5.2 域更新操作指引（必含，产品级必要提示）

依据 tech_full.md 明确结论——python-docx 无法计算域结果，TOC/PAGE/SEQ/REF/STYLEREF 的实际数值需要 Word/WPS 渲染引擎计算——报告与文档交付说明必须包含以下指引，措辞建议原样纳入报告模板：

> **重要：本文档包含自动生成的目录、页码、交叉引用等"域代码"，打开后需手动更新一次才能显示正确内容。**
> - **Microsoft Word**：打开文档后，按 `Ctrl+A` 全选，再按 `F9` 更新所有域；若目录页码未正确显示，可单独右键目录区域选择"更新域"→"更新整个目录"。
> - **WPS Office**：WPS 对域自动更新的支持**不保证**与 Word 一致（已实测确认：WPS 通常不会在打开时自动应用 `w:updateFields` 设置），必须**手动右键目录/页码区域 → "更新域"**。若整篇更新，可尝试全选后 `F9`，但建议以逐个右键更新作为保底方案。
> - 若更新域后仍显示异常，尝试"打印预览"一次（触发 Word/WPS 重新分页计算）后再关闭预览，通常可解决页码/目录不同步问题。

### 5.3 输出格式与落盘

| 输出通道 | 内容 | 格式 |
|---|---|---|
| 控制台 stdout | 精简版摘要（统计摘要 + Fatal/Warning 计数 + 报告文件路径提示） | 纯文本 |
| 文件落盘 | 完整版报告（§5.1 全部6节） | Markdown，UTF-8 编码（无 BOM），与输出 docx 同目录 |

**文件命名**：`<docx文件名(不含扩展名)>-conversion-report.md`（如 `报告题名_v1.0-conversion-report.md`）

### 5.4 警告的可操作性格式（强制规范）

每条 Warning/Info 必须遵循统一格式，禁止只写"存在问题"这类不可定位的描述：

```
[<分级>] 行<N> | <元素类型：图片/表格/标题/交叉引用/分页/字体...> | <问题描述> | 建议动作：<具体可执行的下一步>
```

示例：
```
[WARNING] 行384 | 交叉引用 | 图7-1在行384被预告引用，但实际图片元素在行384之后未找到匹配的"图7-1"题注 | 建议动作：检查图片alt文本"图7-1"拼写是否与正文引用一致
[WARNING] 行506-519 | 表格(附录D图表索引) | 该表位于附录H2下，无独立题注加粗行 | 建议动作：无需处理，附录表格按设计不要求题注（正常场景）
[INFO] H2边界(附录B) | 分页 | 该H2标题前未检测到`---`，已按分页规范化算法自动补插 | 无需处理
```

## 6. 错误处理与降级策略矩阵

| 场景 | 检测时机 | 分级 | 处置 | 报告内容 |
|---|---|---|---|---|
| 图片文件缺失 | 门1 | Warning | 该图跳过渲染，原位置插入占位提示段（如"[图X-Y 图片文件缺失，路径：...]"，样式区别于正文） | 行号+图号+期望路径+建议动作"检查路径拼写或补充图片文件后重新运行" |

| 题注无对应表格（`**表X-Y**`加粗行后未跟表格） | 阶段2解析 | Warning | 该行原样保留为普通段落，不当作表题注渲染 | 行号，标注"疑似孤立题注，未匹配到表格" |
| 表格无题注（附录表） | 阶段2解析 | **正常，不告警** | 若该表所属H2是"附录X："模式 → 按无题注表格渲染，视为设计内正常场景 | 无需报告条目（或仅Info级"附录表按无题注规则渲染"） |
| 表格无题注（正文章节内） | 阶段2解析 | Warning | 与附录表区分：正文H2下的表格若无题注 → 与"正文表应有题注"的既有模式（3/3一致）不符，视为偏差 | 行号，建议动作"确认是否遗漏 `**表X-Y 标题**` 题注行" |
| 编号跳号（图号/表号不连续） | 门2 | Warning | 不修改，原样渲染 | 跳号位置说明（如"图1-1后直接图1-3，缺图1-2"） |
| md 为空 | 门1 | **Fatal** | 中止，产出仅含错误说明的报告 | "输入文件为空，无法转换" |
| md 超大（>10MB阈值） | 门1 | Warning | 不阻断，继续处理 | "文件大小异常（XXX MB），请确认输入完整性" |
| 输出路径不可写 | 阶段0 | **Fatal** | 中止，报告可能无法落盘，退化为控制台报错 | stderr 直接打印"输出路径不可写：<path>" |
| python-docx 抛异常（样式KeyError等） | 阶段3渲染中 | **Fatal** | 捕获异常，不产出半成品文件（先写临时文件，异常时删除临时文件） | "渲染步骤：<步骤名>，涉及元素：<元素定位>，异常：<异常信息>" |
| 图片超大（接近50MB限制） | 门1预检单图 + 门3总量校验 | Warning | 门1对单图>10MB提示；门3对最终文件总量>50MB提示（见§3.3第13项） | 图片路径+文件大小 |
| 密级字样命中（门2解析阶段） | 门2 | **需人工复核**（非阻断） | 自动过滤+高亮复核 | 见 §3.2 |
| 密级字样命中（门3复检，即门2过滤后仍残留） | 门3 | **Fatal** | 中止产出，删除临时文件 | "密级过滤失效：位置<N>，原文：<...>，转换器缺陷，需修复过滤规则" |
| 红队过程标记残留（预处理清理后门2仍检出） | 门2 | Warning（标注转换器缺陷信号） | 不阻断，原样保留（不做二次强制删除，避免误删正文其他含方括号内容） | "预处理规则可能未覆盖此变体，位置<N>，原文：<...>" |
| IR 关键字段缺失（解析器内部问题） | IR完整性自检 | **Fatal** | 中止 | "IR节点<类型>缺失必需字段<字段名>，属于转换器内部错误" |

## 11. D14 域代码分级方案对门禁判定的影响（设计层深化）

D14 裁决"TOC/PAGE 域必做；SEQ/REF/书签题注为推荐实现"。本设计从**门禁判定结果**角度给出分级方案（OXML具体写法归InterfaceDesigner/SystemArchitect）：

| 域类型 | 实现级别 | 若未实现时门3判定结果 | 转换报告"已知限制"提示 |
|---|---|---|---|
| TOC | 必做 | 缺失→Fatal（阻断产出） | 不适用（必须实现） |
| PAGE | 必做 | 缺失→Fatal（阻断产出） | 不适用（必须实现） |
| SEQ（图表自动编号） | 推荐 | 未实现→该检查项从"FAIL"降级为"**N/A（本版本未实现，见已知限制）**"，不阻断 | "图表编号目前为静态文本（非SEQ域），若在Word中调整图表顺序，编号不会自动重排，需重新运行转换器" |
| REF（交叉引用） | 推荐 | 同上 → N/A + 已知限制 | "正文交叉引用为静态文本（非REF域），编号变化不会自动同步" |
| 书签 | 推荐 | 同上 → N/A + 已知限制 | "题注书签未创建，交叉引用无法在Word中Ctrl+点击跳转" |

**设计原则**：SEQ/REF/书签若本版本不实现，**不能**让门3检查表格出现"未评分"的空白项，必须显式标记为 N/A + 已知限制说明，写入转换报告，避免用户误以为是遗漏或缺陷。若实现团队后续决定升级实现，只需将对应行的"未实现"分支替换为 §3.3 同类"A类自动校验"逻辑即可，门禁框架本身不需要改动。

## 12. 退出码语义与状态机

### 12.1 退出码

| 退出码 | 语义 | 触发条件 |
|---|---|---|
| 0 | 完全成功 | 无Fatal；可能有Warning/Info（正常，可用文档已产出） |
| 1 | 转换失败 | 任一阶段触发Fatal（门1空文件/编码错误、输出路径不可写、IR完整性自检失败、渲染异常、门3密级复检命中、门3分页计数断言失败） |
| 2 | 成功但需要人工关注 | 无Fatal，但存在"需人工复核"标记（如密级过滤触发、B类人工复核项非空）——供SKILL.md编排层区分"完全干净成功"与"成功但建议人工看一眼"两种情形 |

（退出码的完整参数化与CLI层规格由InterfaceDesigner定，此处只定语义边界）

### 12.2 状态机

```
[INIT]
   │
   ▼
[门1校验] ──Fatal──→ [ABORT: exit 1, 产出失败报告]
   │Pass(含Warning)
   ▼
[预处理清理] ──(结果为空)──→ [ABORT: exit 1]
   │
   ▼
[IR解析]
   │
   ▼
[IR完整性自检] ──Fatal(内部错误)──→ [ABORT: exit 1, 报告标注"转换器内部错误"]

   │
   ▼
[渲染→临时文件] ──Fatal(python-docx异常)──→ [ABORT: exit 1, 删除临时文件]
   │Pass
   ▼

   │
```
