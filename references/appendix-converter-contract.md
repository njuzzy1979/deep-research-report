# 附录：Markdown 写作规范——与 md→docx 转换器的合约

> 本文件是 deep-research-report skill 的转换器合约附录，从 SKILL.md 附录X 拆分而来。
> 独立文件方便转换器开发者参考，也是阶段9合并终稿前的必备检查依据。
> 母文件：`../SKILL.md`（流程索引）

---

> **为什么需要这份合约**：阶段9的 md→docx 转换器通过解析 Markdown 语法来生成 Word 文档。如果 Markdown 格式不符合转换器的预期，会导致封面标题错误、章节编号混乱、图片丢失、表格题注缺失等问题。以下四条约法是转换器正确工作的前提。

## 合约1：标题层级

| 规则 | 说明 |
|------|------|
| **全文只有一个 H1**（`#`） | 可以是报告主标题或"前言/导论"标识。转换器将第一个 H1 识别为主标题渲染在封面，后续 H1 全部降级处理 |
| **正文各章使用 H2**（`##`） | 不加任何数字前缀或中文数字，只写纯文字标题。转换器自动添加"第X章"编号 |
| **正文各节使用 H3**（`###`） | 不加数字前缀。转换器自动添加"X.Y"编号 |
| **正文各小节使用 H4**（`####`） | 不加数字前缀。转换器自动添加"X.Y.Z"编号 |
| **前言中的 H2/H3 不加数字编号** | 如 `## 问题提出与研究背景`（非 `## 0.1 问题提出`），转换器识别为前置件不参与正文编号 |

**示例对照**：

```
✅ 正确：                      ❌ 错误：
# 前言/导论                    # 0. 前言
## 问题提出与研究背景            ## 0.1 问题提出与研究背景
## 军事需求分析                 # 第一章 军事需求分析
### 非合作目标异动意图判断        ### 1.1 非合作目标异动意图判断
```

## 合约2：图片引用

| 规则 | 说明 |
|------|------|
| **必须使用标准 Markdown 图片语法** | `![图X-Y 标题](相对路径)` |
| **图号使用"章号-序号"格式** | 如 `![图3-1 四层三域总体架构](../figures/3-1-四层三域总体架构.png)` |
| **图片文件必须存在于指定路径** | 转换器会检查文件存在性，缺失的文件触发 ERROR |
| **禁止使用占位符** | 不要写 `[图1-1：xxx]` 或 `图表占位：图1-1` 这类文字占位，转换器无法识别 |

**示例**：
```markdown
![图3-1 系统总体技术架构全景图（四层三域）](../figures/3-1-四层三域总体架构.png)
```

## 合约3：表格题注

| 规则 | 说明 |
|------|------|
| **表格上方使用加粗题注行** | `**表X-Y 标题**` |
| **表格下方使用斜体数据来源行** | `*数据来源：xxx*` |
| **表号使用"章号-序号"格式** | 如 `**表2-1 功能指标**` |

**示例**：
```markdown
**表2-1 功能指标**

| 编号 | 功能指标 | 目标值 |
|------|---------|--------|
| F-01 | ... | ... |

*数据来源：项目技术指标设计*
```

## 合约4：禁止写入的内容

以下内容不应出现在 Markdown 草稿中——它们会干扰转换器的自动编号和渲染：

- ❌ 标题中的手动编号（如 `## 1. 背景`、`## 一、概述`）
- ❌ "建议印刷页数：X页"（大纲阶段的内部标注，不进入正文）
- ❌ HTML 标签（`<div>`、`<span>` 等）
- ❌ "图表占位"标记（如 `[此处插入图1-1]`）
- ❌ "全文完"标记
- ❌ 封面元数据行（如 `编制单位：xxx`，应通过 YAML 元数据块传递）
- ❌ 任何形式的密级标注（"绝密""机密""秘密""内部""内部资料"等）

## 合约自查清单

在进入阶段9合并终稿之前，运行以下检查：

```
grep -n "^#" research/drafts/final-report.md          # 确认只有一个H1
grep -n "!\[图" research/drafts/final-report.md       # 确认所有图片使用标准语法
grep -n "\*\*表" research/drafts/final-report.md      # 确认所有表格有加粗题注
grep -n "建议印刷页数\|全文完\|图表占位\|<div\|密级\|绝密\|机密" research/drafts/final-report.md  # 确认无禁止内容
```

## 封面约定（改进 14：封面独立 MD 文档）

封面信息独立于正文 Markdown 管理，存储在 `research/cover.md`（纯 YAML frontmatter 格式）。

| 规则 | 说明 |
|------|------|
| **封面文件独立于正文** | 封面信息存放在 `research/cover.md`，与正文 Markdown 的 YAML 元数据块分开管理。修改封面只需编辑此文件 |
| **转换器 `--cover` 参数** | `--cover research/cover.md` 指定封面文件路径，封面字段优先级最高 |
| **优先级链** | `cover.md YAML` > `正文 md YAML` > `CLI 参数` > `默认值` |
| **封面文件格式** | 纯 YAML frontmatter（`---` 包裹，不含正文内容） |

**cover.md 格式示例**：

```markdown
---
title: 报告题名
title_en: English Title
report_type: 立项建议书
org: 申报单位
date: 2026年7月
version: V1.0
header_short: 页眉简称
---
```

**字段映射**（cover.md YAML → 封面渲染位置）：

| cover.md 字段 | 封面渲染位置 | 说明 |
|-------------|------------|------|
| `title` | 封面主标题（28pt Bold 居中） | 优先级高于 H1，用于覆盖封面显示标题 |
| `title_en` | 英文副标题（14pt 居中） | 可选，无值时跳过渲染 |
| `report_type` | 报告类型标签（16pt 居中） | 默认"深度研究报告" |
| `org` | 机构名（14pt Bold 居中） | 默认"遨天科技" |
| `date` | 日期（11pt 居中，版本号后） | 如"2026年7月" |
| `version` | 版本号（11pt 居中，日期前） | 如"V1.0" |
| `header_short` | 页眉简称 | 覆盖正文 YAML 的 header_short |

> **注意**：`cover.md` 仅影响封面页渲染和页眉简称，不影响正文标题（TOC 目录项、正文 H1 位置仍使用正文 Markdown 中的 H1 标题文本）。
