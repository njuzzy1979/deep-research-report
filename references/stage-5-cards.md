# 阶段 5：专题研究——卡片与证据包

> 本文件是 deep-research-report skill 的阶段 5 详细 spec，从 SKILL.md 拆分而来。
> 母文件：`../SKILL.md`（流程索引）

---

卡片是**事实核验台账（阶段 3）和正文写作（阶段 7）之间的中间产物**：台账里是零散的、逐条的事实主张，写作时需要的是"一个案例/一项技术/一套架构/一个概念"的完整叙事单元。卡片把台账中相关的主张组织成可直接转写为正文的结构化单元，同时不丢失每条判断背后的证据链。

## 5.1 卡片类型与下游用途

每类卡片对应下游一个明确的使用环节，不是产出后自由存放：

| 卡片类型 | 内容 | 下游用途 |
|---|---|---|
| **案例卡** | 名称、时间线、问题、动作、技术支撑、效果、证据包、风险争议、可提炼机制 | 直接填充 4.2 可选模块中的"案例分析"章节（命中案例研究法时）；每张卡对应案例分析章节的一节 |
| **技术/主题卡** | 核心概念、架构位置、输入输出、关键数据、证据包、与其他组件关系、局限 | 支撑核心分析章节的机制阐述——写作时逐条转写"输入输出/关键数据/局限"为正文的机制描述 |
| **架构卡** | 架构名称、所属层次、组件与关系、数据流、证据包、验证方式、对应的图表 | **阶段 6 出图的直接依据**——"组件与关系""数据流"是绘制架构图的素材来源，不是阶段 4.3 图表清单能替代的细节 |
| **理论/概念卡** | 概念定义、提出者/来源、证据包、应用场景、与其他概念关系、争议 | 支撑前言/导论的概念界定部分，及核心分析中对理论框架的引用 |

## 5.2 证据包——卡片与事实核验台账的绑定方式

**证据包**：卡片中每条关键判断对应的最小引用单元 = `claim_id`（指向 `claims-ledger.csv` 的核验记录）+ 该记录的 `evidence_sources`（指向 `source-index.csv` 的来源）。

不用笼统的"证据等级：中"这样一个印象分描述整张卡片——卡片里通常有 3-5 条不同强度的判断（例如案例卡的"效果"是强判断需要 A/B 级证据，"风险与争议"可能只有 C 级来源），**每条关键判断单独标注证据包**，卡片整体的可信度是这些证据包的最低值，不是平均值或主观估计。

## 5.3 卡片模板（四类，均含证据包字段）

案例卡：
```markdown
## 案例名称
1. **时间线**：
2. **背景问题**：证据包 [claim_id]
3. **关键动作/适变**：证据包 [claim_id]
4. **技术/业务支撑**：证据包 [claim_id]
5. **效果（量化优先）**：证据包 [claim_id]
6. **风险与争议**：证据包 [claim_id]
7. **可提炼机制**：
```

技术/主题卡：
```markdown
## 技术/主题名称
1. **核心概念**：
2. **架构位置**：（在阶段 1.3 分析框架中所属的层次）
3. **输入输出**：证据包 [claim_id]
4. **关键数据**：证据包 [claim_id]
5. **与其他组件关系**：
6. **局限**：证据包 [claim_id]
```

架构卡：
```markdown
## 架构名称
1. **所属层次**：（对应阶段 1.3 分析框架的哪一层）
2. **组件与关系**：（用于阶段 6 出图的核心素材）
3. **数据流**：证据包 [claim_id]
4. **验证方式**：（该架构判断如何被核实，如官方文档/逆向分析/访谈）
5. **对应的图表**：（图号，阶段 6 出图时回填）
```

理论/概念卡：
```markdown
## 概念名称
1. **定义**：证据包 [claim_id]
2. **提出者/来源**：
3. **应用场景**：
4. **与其他概念关系**：
5. **争议**：证据包 [claim_id]
```

## 5.4 卡片索引登记——让"卡片有没有被用上"可追溯

每张卡片产出后，登记到 `research/notes/card-index.csv`（模板见 [`references/card-index-template.csv`](card-index-template.csv)），字段：

`card_id, card_type, title, chapter_ref, evidence_packet_refs, credibility_summary, status, used_in_chapter`

- `card_id`：卡片唯一编号（如 CASE-01、ARCH-02）
- `chapter_ref`：对应大纲中的章节号（写卡片时填，来自阶段 4 大纲）
- `evidence_packet_refs`：该卡片关联的 claim_id 列表（分号分隔）
- `status`：draft / final
- `used_in_chapter`：**阶段 7 写作时回填**——写完一章后，将该章实际引用过的卡片 ID 标记对应章节号；未使用则留空

**先登记再写作**。卡片不登记直接拿去写，等于放弃了阶段 8 检查"哪些调研成果被浪费"的能力。

- [ ] 每张卡片的关键判断均标注了证据包（claim_id），不是笼统的单一证据等级
- [ ] 卡片之间无矛盾（如有矛盾已在卡片中标注）
